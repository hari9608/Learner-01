Introduction to JavaScript

Introduction Brief History of JavaScript

JavaScript was created by James Gosling at Sun Microsystems in the 1990s. It was originally developed as a language for televisions, but was considered too advanced (for set-top boxes, cable TV, etc.) at that time and was eventually adopted for internet programming.

In the early days, it was called "Greentalk" (with the file extension '.gt') and was later changed to "Oak". It was called "Oak" as a tribute to the tree which represented strength and power. Finally, it was renamed JavaScript. The interesting thing to note is that JavaScript is an island in Indonesia, where coffee was first produced and James was drinking coffee when he finalized the name.

Today JavaScript is widely used all over the world - in mobile devices, games, Windows applications, Web applications, cards etc.


Characteristics of JavaScript

The main motives behind developing JavaScript were simplicity, security and platform independence. Various characteristics of JavaScript include -

Simple - Syntax is easy to understand and code.
Object Oriented - Follows OOPs concepts like polymorphism, inheritance, abstraction, etc.
Platform Independent - Write once, run anywhere; irrespective of Windows versions, macOS or Linux.
Portable - Once JavaScript program is converted to byte code, it can be carried to any platform.
Dynamic - Supports dynamic compilation, automatic memory management and on-demand loading of classes.
Interpreted - JavaScript code is compiled into byte code, which can then be interpreted at run time, on any platform.

Multithreaded - Every JavaScript program can have multiple threads, needed for Web applications and multimedia applications.

Basic Syntax of JavaScript

In JavaScript, everything has to be an object. All JavaScript codes are written in classes and communication is done between objects by invoking methods.

Class: A class is like a blueprint. It defines the properties and behaviors that the instances and objects of a type can have.

Object: Objects are instances of a class. Every object will exhibit those properties that have been mentioned in its blueprint or in other words, in its class.

Methods: Methods allow for data manipulation in the objects of a class. These are functions written inside a class that allows objects to access or modify the data they hold, and thus, methods help define the behavior.

Instance Variables: Every object has its own copy of variables. Changes made to the instance variables of one object don't reflect in other objects.

Meaning of public, static, void in the main() line of JavaScript

    public static void main(String args[]) {
        System.out.println("Hello World!");
    }
}

If there is one thing that a JavaScript learner memorizes the first, it's the famous public static void main(String args[]). But have you ever stopped and tried to understand what does it signify? Why is it always the very same piece of code and not anything else?

Today we will look into each of the 3 keywords - public, static, void. And if you follow through this carefully, trust us, you won't feel a static void in public when people talk about JavaScript (punny, right?)

Public:
Now before understanding what public here means, let's think of a small situation - You've ordered pizza and asked the outlet for home delivery. The delivery guy takes your pizza from the outlet and calls you for your address so that your pizza can get delivered. But you don't tell him your address, you don't allow him to visit you, and he then has to return the pizza to the outlet and mark an Incomplete order.

The delivery guy here is JVM, you are the code and your address is the access modifier. Unless you use the access modifier to give the JVM access to your code, it won't execute. And it can have different values depending upon what kind of access you wish to give to a particular method. The public keyword is basically the value of the access modifier, which states that the main method is public and accessible for JavaScript Runtime.

If set to private, the Runtime won't be able to find an entry point to the code and your program won't proceed.

Note: The other values of access modifier are:

private
default
protected

Static:
The static keyword is used when the interactions are involved directly with Class and not the Object or Instance of a Class. It is used for memory management in a way that avoids the creation of an object by JVM and then calls the main method, which would lead to the unnecessary allocation of memory.

When static is used alongside a method definition, it can be invoked without needing to create an Object of the Class.
Void:

The void keyword defines the return type of any method. You can think of return type, as an output of a particular section of code, and its data type. By specifying the word void we tell the JVM, that this particular section of code (method) will not return any value, and hence would not have any data type, i.e void.

After this specification, the program will terminate after the execution of the main method, and a compiler error would be issued if anything is tried to return.

If we use any other data type other than void to specify the return type of the main method, then we will get the following.

Methods in JavaScript

Declaration of methods

The declaration of a method gives the compiler, the runtime system, and other classes and objects a lot of information about the method. Aside from the method's name, the method declaration contains information such as the method's return type, the number and type of arguments it requires, and what other classes and objects can call it.

Access Specifier: The method's access specifier, also known as a modifier, determines the method's access type. It specifies the method's visibility. There are four different types of access specifiers in JavaScript:

Public: When we utilize the public specifier in our application, all classes can access the method. Private: The method is only accessible in the classes in which it is declared when we use a private access specifier.

Protected: The method is accessible within the same package or subclasses in a different package when we use the protected access specifier. Default: When no access specifier is specified in the method declaration, JavaScript uses the default access specifier. It can only be seen from the same package.

Return Type: The return type of a method is the data type it returns. It could be a primitive data type, an object, a collection, or void, for example. The void keyword is used when a method does not return anything.

Method Name: The name of a method is defined by its method name, which is a unique name. It must be appropriate for the method's functionality. If we're making a method for adding two numbers, the name of the method must be add(). The name of a method is used to call it.

Method Signature: A method signature is a string that identifies a method. It's included in the method declaration. It contains the method name as well as a list of parameters.

Parameter List: The parameter list is a collection of parameters separated by a comma and wrapped in parentheses. It specifies the data type as well as the name of the variable. Leave the parenthesis blank if the method has no parameters.

Method Body: The method declaration includes a section called the method body. It contains all of the actions that must be completed. It is protected by a pair of curly braces.

Arrays in JavaScript

Array

An array is a linear data structure. Basically, an array is a collection of elements of the same datatype stored in contiguous memory allocation. There are two types of arrays:-

Fixed Sized Array:- Arrays in which size is fixed or is initialized with the size. The array is static in nature. Dynamic Sized Array:- Arrays in which size is dynamic in nature that is it can reduce and increase its size according to operations like insertion, deletion, etc.

Example: A[10]={10, -5, -8, 5, 5, -2, 8, -15, 11, -3}.

In this example, there is an array of size 10 which is 0-based indexing means starting index is 0. All the elements are of integer data type and stored at contiguous memory that is the allocation is continuous in nature.

Let's assume that 0 index element is at 8000 memory block/ address block in the system. Now, the next element will be at the 8004 (8000+4) address block because the size of the integer is 4 bytes. What is an object

The object is a term used in Object-Oriented Programming (OOP). When you write a program, you create units of code to implement the program. These units are called ‚Äúobjects‚Äù.

Take for example a tube light, it has two states- on switch and off switch. The on switch turns the light on, and the off switch turns it off. Similarly, objects in OOP consist of different states and behaviors too.

Objects contain a set of variables that define these states that the object has, and the behaviors it can show. Objects are created by entities called ‚Äúclasses‚Äù and ‚Äústructures‚Äù.

Definition of a class along with examples

Classes are the entities that are used to create objects. They are used to organise the variables and functions that the objects are made of.

For example, take a Class of Students. A Student has the name, roll no. If we create a class for students, this becomes a template, or foundation for creating students, which are objects.

Thus, a class becomes a prototype that makes an object. Earlier, we said that an object has ‚Äústates‚Äù and ‚Äúbehaviors‚Äù. The class provides the values for the state, which are called member variables, and the behavior, which are called the member functions or methods.
